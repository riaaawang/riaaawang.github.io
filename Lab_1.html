Lab Report 1 - Artemis, and build Github Page

The objective of this lab was to provide an introduction to programming the Sparkfun Artemis, a powerful microcontroller board that is widely used in the field of robotics. The lab consisted of setting up the Arduino IDE and running example code to test basic functionality. This report will detail the four examples that were completed during the lab, including Blink, Serial, Analog Read, and Microphone Output.

Example 1: Blink It Up

https://youtube.com/shorts/LV7YbvSBOMw

The first example involved testing the LED on the Artemis board by blinking it using the "Blink" example from the Arduino IDE. The LED_BUILTIN pin in the code was mapped to the correct pin on the Artemis. A 1-second on and 1-second off pattern was used to confirm the basic functionality of the LED.

Example 2: Serial

https://youtube.com/shorts/ZysTG3YbnTE?feature=share

The second example involved running the Artemis Serial example code to test the serial port. The baud rate was set to 115200, which is a common rate used in many serial communication applications. The serial monitor was used to confirm that the text typed in was echoed back, indicating that serial communication was functioning properly. This example demonstrated how to set up basic input and output functionality over a serial port.

Example 3: Analog Read

https://youtube.com/shorts/taM7BH0kLmI

The third example involved using the Artemis Analog Read example code to read the onboard temperature sensor. The temperature reading began at approximately 31,800 and increased to over 32,000 when the sensor was exposed to the heat release vent of a laptop, confirming the functioning of the sensor and the analog-to-digital converter (ADC). This example demonstrated how to read analog values from sensors and use them in your program.

Example 4: Microphone

Figure 2 shows the reaction of my board during not 430 Hz

As part of the lab exercise, I utilized the Example1_MicrophoneOutput code under the PDM of Arduino Example to test the onboard microphone of the Sparkfun Artemis board. The code was designed to print out the loudest frequency after performing a Fast Fourier Transform (FFT) on the audio signal captured by the microphone.

During testing, I decided to extend the functionality of the code by incorporating a visual indicator that would turn on and off based on the frequency of the audio signal detected by the microphone. Specifically, I added code to the printLoudest function that would light up the blue LED on the board when an A note (which ideally has a frequency of around 440hz) was played over the speaker. The intention was to use the LED as a simple visual cue to help distinguish different audio signals and frequencies.

However, during testing, I discovered that the board recognized the frequency of the A note I played as 434 hz instead of the expected 440hz. This deviation in frequency could be attributed to factors such as the performance of the microphone and the FFT algorithm used in the code.

To account for this deviation, I modified the condition in the code that lights up the blue LED to frequencies under 434hz, instead of the expected 440hz. Upon testing, I found that the modification worked as expected and the LED lit up when I played notes in the appropriate frequency range, and turned off otherwise.
